Modern web-application functionality without leaving the comfort of HTML.

What is it?

Experience UI is a suite of custom HTML elements that turns a static HTML file into single-page web-applications experiences.

This documentation was created with these experience elements. View-source or take the tour to see how complex routing, guided navigation, automatic presentations, data-binding and more is made possible!

Turbo-charged HTML

Content routing, guided navigation, data-binding, dynamic content, and even event-based actions are simple to add to any page with little scripting an no framework. PLUS, our convention-over-configuration approach dramatically simplifies complex functionality.

These elements offer application functionality for websites built without a web-application.

How does this help?

Frameworks exist to minimize the effort to create and maintain web pages. They do that well, but at a high costs and long-term commitment not suited for simple use-cases.

To get these time-saving features, developers must choose e framework or static site generation tool. These tools are often complex and time-consuming to learn, set up and operate. Once a framework is adopted, you are locked-in and it's difficult to change direction, without a costly endeavor.

No Framework Necessary

<x-ui/> is different. Without transpilation, a framework or even a build-process, developers get the same time-saving techniques (layouts, routing, data-binding, etc) using specialized elements directly in the HTML.

There is no need to check if your site is compatible -- if you can include a <script> tag, you can start turn your static HTML site into developer-friend web-application today.

Features:

  • Application-like Routes w/Lazy Loaded HTML
  • Support for Data Routes for Data-Driven Content
  • Use Route Data Directly in HTML w/Route and URL Data Providers

  • Automatic Back & Forward Resolution
  • Sequenced Content: present, educate, inform
  • Sequenced Input: surveys, quiz and display results
  • Sequenced Video: interactive video story-telling
  • Declarative: Expression-based Conditions
  • Stateful: pick up where a user left-off

  • Data Tags to render personalized data from a variety of sources
  • Use HTML to access data from your API
  • Capture attention and data, with smart workflows to replace boring forms.
  • Personalize the content for each of your visitors.

  • Interactive Videos
  • Queued, Timed & Event-based Actions
  • Animated Transitions
  • Background Music & Voice-Overs
  • Animation Hooks

What it is not.

<x-ui/> experience elements are not a UI Framework. These elements work with the UI framework of your choice and offer no opinions on how your content should look.
