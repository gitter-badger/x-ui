What is <x-ui/>?

view.DO's <x-ui/> is the first UI component toolkit built specifically for turning static content into interactive web experiences.

How does this help?

For most developers, building a web-app begins by choosing an app framework or static site generation tool. These tools are often complex and time-consuming to learn, set up and operate. Once a framework is adopted, you are locked-in and it's difficult to change direction, without a costly endeavor.

Frameworks exist to minimize the effort to create and maintain web pages. There are many tools that do this well, but they all come with a huge cost and a long-term commitment.

No Framework

<x-ui/> is different. Without interpolation, a framework or even a build-process, developers get the same time-saving techniques using web components directly in HTML. Better yet, it can be added to any site or page with a simple script reference. Have a framework already? No problem!

<x-ui/> components make it simple! We created special elements for expressing routes, navigation, dynamic-content, data-binding and much more. We use a convention-over-configuration approach to take simplify things freeing you up to focus on the content.

It's not magic, it's view.DO!
