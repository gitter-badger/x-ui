Routing

Content routing is a simple declarative approach for creating multi-page applications from a single HTML file. The browsers URL path will determine what content is be displayed.

How does it work?

By wrapping content in an <x-view> or <x-view-do> elements and setting the url, the contained-content won't be displayed until the URL path matches the path set in the url attribute.

These elements work very similarly to typical web frameworks, minus the framework. You can organize HTML fragments as separate files where it makes sense.
