Routing

Content routing is a simple declarative approach for creating multi-page applications from a single HTML file, using the browser's URL path to determine what content should be displayed.

A Simple Router

Routing doesn't need to be complex. URLs are the best way for identifying and locating content, but until now - you had to choose between development efficiency and production complexity. Have your HTML-only cake, and eat it too.

See Examples

<x-ui/> supports most real-world routing needs and takes further by using HTML to construct semantic route rules. Check out how easy it is to handle advanced routing needs.

How does it work?

<x-ui/> works very similarly to typical web frameworks, minus the framework. You can organize HTML fragments as separate files where it makes sense.

By wrapping content in an <x-view> or <x-view-do> elements and setting the url, the contained-content won't be displayed until the URL path matches the path set in the url attribute.
