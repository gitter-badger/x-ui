Routing

Lazy Loaded HTML

Route content could be inline or fetched from remote files, or "lazy-loaded" (as needed). The home-page could be inline, while all other routes are lazy-loaded. This approach gives you the best of both static sites and dynamic sites, keeping the initial page size light-weight and progressively fetching HTML fragments only when needed.

The <x-include> component fetches remote HTML and renders it safely and directly into the page when when and where you tell it too.

You get the benefit of shared layouts and components without the need for a application-server, client-side framework or static-site generation process.
